## 1. 앱 (Flutter) – 미구현/차이

### A. 온보딩
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 서비스 컨셉·효과·사용 흐름 소개 | ✅ 3페이지 온보딩 있음 |
| 스킵 가능 | ✅ "건너뛰기" 있음 |
| **재진입 시 노출 여부 설정** | ❌ 미구현 (한 번 로그인 후 온보딩 다시 보기 옵션 없음) |

### B. 회원가입 / 로그인
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 구글/애플 로그인, 로그인 상태 유지, 로그아웃 | ✅ 구현됨 |
| **회원 탈퇴** | ❌ 미구현 (계정 화면에 탈퇴 진입/API 연동 없음, 문서상 DELETE /me는 명세만 존재) |
| 재설치 후 계정 복구, 다기기 동기화 | ✅ sync 구조 있음 |

### C. 메인 홈 / 히트맵
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 오늘의 습관 목록, 진행 상태, 달성/미달성, 연속 달성 | ✅ 구현됨 |
| 히트맵 표시 | ✅ 28일 히트맵(_HeatmapSection) 있음 |
| **레벨 또는 포인트 노출** | △ 홈에 _LevelProgressCard로 "LVL 1~3" 표시하나, **서버 GET /me/level API 없음** (클라이언트 로직만 존재) |
| 감성 피드백 애니메이션 | △ 완료 시 다이얼로그만 있음, **Lottie/햅틱/사운드 미적용** |

### D. 습관 등록 UX
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 습관명, 카테고리, 시작일, 알림 여부·시간 | ✅ 이름·카테고리·알림 있음. 시작일은 **오늘 고정** (날짜 선택 없음) |
| **목표 유형 설정** (완료 여부형/횟수형/시간형/수치형) | ❌ 앱에서 선택 불가. 서버·로컬엔 goalType 필드 있으나 습관 생성 시 'completion' 고정 |
| **반복 주기 설정** | ❌ 미구현 (HabitSchedule 엔티티/API 없음) |
| **색상/아이콘/캐릭터** | ❌ 습관 생성·상세에서 선택/수정 불가. 서버·로컬엔 colorHex, iconName 필드만 존재 |
| 사전 제공 습관 템플릿 | ❌ 앱에서 템플릿 선택해 습관 만드는 흐름 없음 (관리자 템플릿 CRUD만 존재) |

### E. 습관 기록 / AI
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 습관별 기록, 완료 직후 AI 코멘트, fallback, 호출 제한 | ✅ 구현됨 |
| 기록 수정 가능 여부 | △ 기록 수정 UI 없음 (추가만 가능) |

### F. 통계 / 레벨업
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| **일/주/월 기준 통계 조회** | ❌ 일/주/월 탭 없음. 오늘 요약·습관별 연속일·AI 피드백만 있음 |
| 달성률, 연속 달성일, 습관별 성과 | △ 연속일만 표시. **달성률·기간별 집계 없음** |
| **레벨/포인트 시스템** (서버 산정) | ❌ GET /me/level 미구현. 홈 레벨은 클라이언트 단순 계산만 |
| 성장 피드백 메시지 | △ 레벨 카드 문구만 있음 |
| **히트맵 산식·레벨업 조건·누락 기준** | PRD 오픈 이슈대로 별도 정책 문서·구현 없음 |

### G. 상세 습관 설정
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| 습관명 수정, 목표 수정, 반복 주기, 알림 시간 | △ 알림만 상세에서 변경 가능. **습관명/목표/반복 주기 수정 UI 없음** |
| **아이콘/테마(색상) 수정** | ❌ 미구현 |
| **아카이브** | ❌ 삭제만 있음. 보관(archivedAt) API는 서버에 있으나 앱에서 아카이브 진입 없음 |
| 삭제 | ✅ 구현됨 |
| 기록 히스토리 확인 | ❌ 습관별 기록 목록/캘린더 뷰 없음 |

### H. 전체 설정

| **문의하기** | ❌ 미구현 |

## 2. 관리자 – 미구현/차이

### 메뉴 구조 (PRD 9.2)
| PRD 메뉴 | 현재 상태 |
|----------|----------|
| 관리자 로그인 | ✅ 구현됨 |
| 대시보드 | ✅ 가입 회원/전체 습관/전체 기록, 회원당 평균 등 |
| 회원 관리 | ✅ 목록, 가입일/습관 수/완료율 포함 |
| 습관 템플릿 관리 | ✅ 구현됨 |
| **습관 카테고리 관리** | ✅ 구현됨 (PRD에는 "습관 템플릿"만 명시, 카테고리는 추가 구현) |
| 공지/운영 콘텐츠 관리 | ✅ 구현됨 |
| AI 문구/설정 관리 | ✅ SystemConfig에 AI fallback, 일일 상한, JWT 만료 등 |
| **통계 조회** | △ 대시보드에 기본 지표만 있음. **별도 "통계 조회" 전용 페이지/기간별·상세 통계 없음** |
| 시스템 설정 | ✅ system-config로 키/값 관리 |

### 상세
| PRD 요구사항 | 현재 상태 |
|-------------|----------|
| AI **프롬프트** 관리 | △ fallback 문구·일일 상한만 있음. **프롬프트 문구 직접 수정 기능 없음** (서버 하드코딩) |
| 기본 통계 대시보드 | ✅ 구현됨 |
| 설정값 관리 | ✅ system-config로 확장 가능 |

## 3. 서버/API – 미구현/차이

| 항목 | 현재 상태 |
|------|----------|
| **GET /me/level** (레벨·포인트) | ❌ 엔드포인트 없음 (api_endpoints에 경로만 존재) |
| **DELETE /me** (회원 탈퇴) | ❌ 구현 없음 (명세만 문서에 있음) |
| **HabitSchedule** / 반복 주기 | ❌ 엔티티·API 없음. 습관은 startDate만 있음 |
| **UserLevel** (또는 레벨 산정 로직) | ❌ 엔티티/서비스 없음 |
| **NotificationSetting** (전역 알림 설정) | ❌ 서버 모델/API 없음. 앱은 습관별 로컬 리마인더만 사용 |
| **WidgetState** | ❌ 엔티티/API 없음 |

| **P2** | 위젯 | iOS/Android 홈 위젯 1종 이상 |
| **P2** | 감성 UX | Lottie, 햅틱, 사운드 |
| **P2** | 습관 상세 | 습관명/목표/반복 수정, 아카이브, 아이콘/색상 수정, 기록 히스토리 |

| **P3** | 데이터/서버 | HabitSchedule, UserLevel, NotificationSetting, WidgetState 등 정책 확정 후 구현 |

## 5. 참고

- **이미 구현된 PRD 항목**: 온보딩(스킵), 구글/애플 로그인, 습관 CRUD, 습관 기록, 홈·기본 통계·히트맵, AI 단발 코멘트, sync/복구, 기본 설정(계정·로그아웃), 관리자 로그인·회원·템플릿·공지·시스템설정·대시보드, 습관 카테고리(관리자+앱).
- **정책/산식 미정**: 통계·레벨업 산정, 히트맵 산식, 연속 끊김/누락 기준, 알림 조건·우선순위, 동기화 충돌 정책 등은 PRD 17. 오픈 이슈에 따라 별도 문서 확정 후 반영하는 것이 좋음.
