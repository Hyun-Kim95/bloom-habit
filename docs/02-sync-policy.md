# 동기화 정책 문서 (초안)

**버전:** 0.1  
**작성일:** 2026-03-17  
**상태:** 초안 (개발 중 협의·조정 가능)

---

## 1. 목적

- 앱(로컬)과 서버 간 **데이터 동기화 시점·방향·충돌 해결** 기준을 정의한다.
- 재설치·다기기·오프라인 등 시나리오에서 **데이터 일관성과 복구**를 보장한다.

---

## 2. 전제

- **단일 소스 of truth**: 습관·기록 등 **핵심 데이터는 서버가 원본**이다.
- **로컬**: 오프라인 사용·빠른 UI를 위한 **캐시**이며, 로그인 사용자는 주기적·이벤트 기반으로 서버와 동기화한다.
- **비회원**: 로컬만 사용(정책 확정 시). 회원 전환 시 업로드 정책 별도 정의.

---

## 3. 동기화 시점

### 3.1 최초 로그인 시

- 로그인 성공 후 **전량 풀(pull)**:
  - 사용자 프로필, 습관 목록, 습관별 스케줄, **기록(habit records)**.
- 로컬 DB에 **upsert** (있으면 갱신, 없으면 삽입).
- 이후 **앱 기동 시** 또는 **포그라운드 복귀 시** 증분 동기화 가능 (타임스탬프 기반).

### 3.2 재설치 후 복구 시

- 사용자가 **동일 계정으로 로그인**.
- 서버에 있는 해당 사용자 데이터를 **전량 풀**하여 로컬 복구.
- “마지막 동기화 시각” 등 메타는 서버 기준으로 초기화.

### 3.3 다기기 접속 시

- 각 기기는 **자체 로컬 DB** 유지.
- 로그인 시·주기 동기화 시 **서버 ↔ 해당 기기**만 동기화.
- **서버가 최신 집계의 원본**이므로, 다른 기기에서 넣은 기록도 풀 시 반영.

### 3.4 오프라인 후 재연결 시

- **재연결 감지** 후:
  1. 로컬에서 **서버로 미동기 변경사항 푸시** (있는 경우).
  2. **서버 → 로컬** 풀로 최신 상태 반영.
- 충돌 시 아래 “충돌 해결” 정책 적용.

### 3.5 삭제·탈퇴 시

- **습관/기록 삭제**: 삭제 플래그 또는 삭제 이벤트를 서버에 반영 후, 동기화 시 다른 기기·재설치 복구 시에도 삭제 상태 유지.
- **회원 탈퇴**: 서버에서 사용자·관련 데이터 삭제(또는 익명화) 정책에 따름. 로컬은 로그아웃 시 정리 또는 앱 재설치 시 초기화.

---

## 4. 동기화 방향

| 방향 | 시점 | 내용 |
|------|------|------|
| **서버 → 로컬 (Pull)** | 로그인 직후, 주기, 재연결 후 | 사용자·습관·스케줄·기록·통계 캐시 |
| **로컬 → 서버 (Push)** | 기록 생성/수정/삭제 시, 앱 종료/백그라운드 전 등 | 새 기록, 수정/삭제 이벤트 |

- **Pull**: “last_synced_at” 또는 “updated_at” 기준으로 증분 가능.
- **Push**: 생성/수정/삭제 시 즉시 또는 큐에 쌓아 배치로 전송.

---

## 5. 충돌 해결

### 5.1 원칙

- **같은 엔티티(동일 ID)에 대한 동시 수정**이 있을 때: **Last-Write-Wins (LWW)**.
- 각 레코드에 **updated_at** (서버·로컬 모두)을 두고, **값이 더 최신인 쪽**을 채택.

### 5.2 적용 단위

- **Habit**: 습관 수정. LWW.
- **HabitRecord**: 일자+습관이 같으면 “해당 일 기록” 1건으로 봄.  
  - 동일 일자에 로컬·서버 각각 기록이 있으면 **updated_at 기준** LWW.  
  - 또는 “기록”은 **추가만 하고 수정/삭제 제한”** 정책 시 충돌 최소화.

### 5.3 삭제 충돌

- 한쪽에서 삭제, 다른 쪽에서 수정: **삭제를 우선**할지, **updated_at 기준**할지 정책 확정.
- 초안: **삭제 요청이 서버에 반영되면 삭제 우선**. 로컬에만 삭제가 있고 서버에 수정이 있으면, 풀 시 서버 상태로 덮어쓰기(삭제 취소).

---

## 6. 메타데이터

- **로컬**: `last_synced_at`, 필요 시 `pending_push_count` 등.
- **서버**: 각 테이블 `updated_at` (또는 `synced_at`).
- 클라이언트는 **증분 풀** 요청 시 `?since=last_synced_at` 형태로 전달.

---

## 7. 오류 처리

- **네트워크 오류**: 로컬 큐에 푸시 이벤트 유지, 재시도(백오프).
- **인증 만료**: 재로그인 유도 후 동기화 재시도.
- **409 Conflict**: 서버에서 충돌 반환 시, 클라이언트는 최신 서버 데이터로 로컬 갱신 후 필요 시 사용자 재입력 유도.
