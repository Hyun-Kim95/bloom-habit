# 계정·비회원 정책 (초안)

**버전:** 0.1  
**작성일:** 2026-03-17  
**상태:** 초안 (개발 중 협의·조정 가능)

---

## 1. 목적

- **비회원 모드 지원 여부**, **탈퇴 후 데이터 보관 기간**, **동일 이메일 중복** 처리 방향을 정리한다.

---

## 2. 비회원 모드

### 2.1 결정: **1차에서는 미지원**

- 앱 사용을 위해 **로그인(구글/애플)** 이 필요하다.
- 이유: 데이터 복구·다기기 동기화·관리자 운영을 단순화하고, PRD의 “계정 기반 복구”를 일관되게 적용하기 위함.

### 2.2 향후 검토

- “체험만 하기” 등 **비회원 모드**를 도입할 경우:
  - 로컬 전용 저장.
  - 회원 전환 시 **로컬 데이터 업로드** 흐름 및 정책 별도 정의.

---

## 3. 동일 이메일/계정 중복

### 3.1 소셜 로그인 1:1

- **구글/애플 각각** 하나의 계정으로 한 사용자와 1:1 매핑.
- 동일 이메일이라도 **구글으로 가입한 계정**과 **애플로 가입한 계정**은 **별도 사용자**로 둘지, **이메일 기준 통합**할지 결정 필요.
- **초안**: **이메일 기준 통합**.  
  - 최초 로그인 시 `AuthAccount`에 provider+uid 저장.  
  - 동일 이메일로 다른 provider 로그인 시 **기존 사용자에 provider만 추가** (동일 사용자로 취급).

### 3.2 중복 가입 방지

- 동일 (provider, provider_uid) 조합은 **한 계정만** 생성. 재로그인 시 기존 계정으로 로그인 처리.

---

## 4. 탈퇴 후 데이터 보관 기간

### 4.1 결정: **탈퇴 즉시 삭제(또는 익명화)**

- **보관 기간**: **0일**. 탈퇴 처리 시점에:
  - 사용자 계정·습관·기록·AI 로그 등 **개인 연계 데이터 삭제** 또는 **익명화**.
- 법적 의무 보관(예: 거래 내역)이 없는 한 **즉시 삭제**로 일원화.

### 4.2 복구 불가

- 탈퇴 후 **동일 소셜 계정으로 재가입** 시 **새 사용자**로 생성. 과거 데이터 복구 없음.

### 4.3 예외

- 법·정책상 보존 기간이 정해진 항목이 있으면, 해당 기간만 익명화 보관 후 삭제. (현재 서비스 범위에서는 해당 없음으로 가정.)
